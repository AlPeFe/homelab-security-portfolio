Homelab Security Architecture

Zero inbound ports · Zero Trust edge · Defense in depth

Security Note: No inbound ports are open on the home router. All external traffic enters exclusively via Cloudflare Tunnel (outbound-initiated persistent connection). Router NAT table only sees egress. Verified via external nmap scan post-deployment.

Threats Mitigated

  • Direct port exposure eliminated — Zero open ports at router level remove entire classes of network-layer attacks (CVE exploitation, brute-force, port scanning).
  • Credential stuffing reduced — OAuth2 via Google shifts credential risk to an IdP with MFA, brute-force detection, and breach monitoring.
  • Geographic attack surface collapsed — EU-only filter blocked >90% of automated scanning traffic in the first week.
  • Service enumeration prevented — Separate subdomains do not reveal IP or topology; origin is fully masked by Cloudflare.

Zero Trust Principles Applied

  • Never trust, always verify — Every request to admin tools is re-authenticated via OAuth2, regardless of previous access.
  • Assume breach — Segmented hosts mean compromise of one service does not grant lateral movement to the other.
  • Least privilege — Tailscale ACL grants per-device and per-port; media services have no write/admin capabilities.
  • Verify explicitly — Cloudflare Access logs every identity decision; Tailscale audit log captures every network flow authorization.

Access Control Matrix

  • Komga / Navidrome — Public subdomains, native basic auth. Audience: media consumers (friends, family).
  • IT Tools / BentoPDF — Public subdomains, OAuth2 enforced before packet reaches origin. Audience: admin only.
  • Server A (Hermes, n8n) — No public exposure. Accessible only via Tailscale VPN from ACL-authorized devices.
  • Portainer — No public exposure. Docker socket access restricted to Tailscale-authorized admin devices.
